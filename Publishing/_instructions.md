# Publishing — Management Instructions

PURPOSE: Claude's guide to the user's publishing workflow. Hybrid role:
enough substance to answer simple/moderate questions directly, plus
source-video references so Claude can route the user back to the right
material when deeper detail is needed.

The video is the deliverable — notes exist to help Claude recognize
what a source covers and when to forward the user to rewatch it.

WHAT TO SAVE (per video/source)
  title, URL, topic scope
  key decisions/options the source covers (enough to answer questions)
  "forward here when..." routing trigger
  tool names, platform names, recommended workflows

WHAT NOT TO SAVE
  step-by-step tutorials (note that they exist in the video, don't reproduce them)
  exhaustive feature lists
  general publishing knowledge Claude already has implicitly

CONSTRUCTION
  one file per publishing topic cluster (ebook formatting, print, distribution, etc.)
  source block per video at the top of each entry
  below source block: key knowledge in labeled list or plain prose
  aim for scannable — Claude needs to route quickly, not read densely
  ~30–50 lines per file

LAYOUT
  add to existing file when a new video fits a topic already covered
  new file when a video introduces a distinct publishing topic
  create subfolders if the folder grows large (e.g. Publishing/Marketing/)
