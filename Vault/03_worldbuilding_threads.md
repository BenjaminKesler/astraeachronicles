# Worldbuilding Threads — generative backlog

A never-exhausted, prioritized store of worldbuilding **threads to develop**, so the good ones don't evaporate as the list grows. Worldbuilding is generative and unbounded; this is the parking lot for "areas we could build out," sorted by importance.

**What goes here vs. [[01_open_items]]:** `01_open_items` tracks *questions that need an answer* — a specific unknown or fork that blocks drafting (resolve, then delete). This tracks *areas to expand* — generative directions that grow and never strictly "resolve." Binary gap to fill → `01_open_items`. Open direction to grow → here. A thread here may spawn a question there.

**Format:** flat list, each line = `[P#] [Tag] thread — optional one-line hook`, where `P#` is priority **1–10 (10 = most important)**. The `[Tag]` is optional, for slicing once the list gets long. Re-score freely — this is a *store, not a queue*; a high-priority thread can sit unworked if it's premature. *(Tags + hooks are a light add over the plain flat-list spec — drop them if they're friction.)*
