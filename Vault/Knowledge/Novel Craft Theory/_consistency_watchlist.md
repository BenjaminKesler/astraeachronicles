HOW TO USE
  - each external case: log which seams it put under load and how the text ruled — "S#: outcome".
  - promote CALL → RESOLVED only when a real arc/text forces it; never resolve in the abstract.
  - S1 first: if "identity" turns out to be "abstract desire" under another name, S1 and part of S4 collapse together.
  - when a SYNC item is confirmed, the fix is an edit to wound-model-thesis, not a model change.
  - CASE SELECTION: S1/S4 live at the aha — only a story whose transformation LANDS can arbitrate them;
    a story that fails before the aha validates the diagnostic CORE but cannot reach the aha-side seams.
  - BEND vs ARCHITECTURE (interpreting deviations): the model describes LOAD-PATHS, not a fixed sequence. A deviation
    is a BEND if it's derivable from existing pieces re-ordered / re-proportioned (no new mechanism); it's a SEPARATE
    ARCHITECTURE only if it requires a load-path the model genuinely can't produce. Reserve new arc-paths for the latter,
    or the model bloats. ("Bend not break" = re-order/re-proportion freely; never DELETE a load-bearing path — that's Parno.)
    worked example: Iron Man front-loading = derivable (compressed early Disaster/Dark/Aha + stretched proof) = BEND, not architecture (C4).
  - VEILED-QUESTION PROTOCOL (for "feel" readers / hazy recall): instead of one open gut-read pass, pull the read with
    CONCRETE, EXPERIENTIAL, BROAD questions (low point / turn / landing — "what's the voice in his head," "what does the
    shift feel like") that never name the axis being tested. Keeps the reader blind to the target AND keeps Claude from
    cherry-picking confirming moments. Strongest evidence yet (C5): the reader can't aim at an axis they can't see.
