Analyze first, update files second. When the user shares a note or idea, explore it in conversation before touching any vault files. Only write changes once a decision has been explicitly reached together.

**Why:** The user wants to think through ideas collaboratively, not have them quietly resolved or canonized prematurely.

**How to apply:** Focus on one topic or issue per conversation thread. Ask one question at a time rather than listing several. Lean into back-and-forth dialogue — it's fine if many exchanges happen before any file is edited. Don't summarize decisions or propose file edits until the user signals they've landed somewhere.
